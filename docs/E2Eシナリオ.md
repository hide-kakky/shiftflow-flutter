# SHIFTFLOW E2E Scenarios v1.1

## 1. 使い方
- 各シナリオを iOS / Android / Web で最低1回ずつ実施する。
- 結果は `pass/fail` とログURLまたはスクリーンショットを残す。

## 2. シナリオ
### E2E-01 ログインとセッション復元
1. Auth画面でログイン
2. 状態に応じて `参加前` または主要画面へ遷移
3. active 状態でアプリを再起動
4. 認証画面を経由せず元の業務シェルへ戻る

### E2E-02 組織コード参加申請
1. 未参加ユーザーでログイン
2. 組織コード検索
3. 参加申請
4. 承認待ち画面表示

### E2E-03 招待リンク参加
1. 招待リンクを開く
2. ログイン
3. 対象組織へ参加
4. 主要画面へ遷移

### E2E-04 現在地ユニット切替
1. Messages を開く
2. 上部文脈切替からユニット変更
3. 表示一覧と件数が変わることを確認

### E2E-05 上位/下位ユニット導線
1. `上位ユニット` タブ表示
2. `下位ユニット` タブ表示
3. 子ユニットカードから現在地変更

### E2E-06 個人メッセージ
1. 個人メッセージ作成
2. 受信側で通知と一覧反映確認
3. 共有メッセージ一覧と混線しないことを確認

### E2E-07 タスク作成〜完了
1. Tasks で新規作成
2. `unit` 文脈付きで保存
3. 一覧反映
4. 完了

### E2E-08 管理画面
1. モバイルで段階表示確認
2. PCで分割表示確認
3. ユーザー更新 / 組織更新 / 監査ログ確認

### E2E-09 誤操作防止
1. メッセージ投稿前に投稿先確認
2. 削除時に対象名付き確認
3. 権限変更時に影響表示

### E2E-10 添付
1. タスク添付アップロード
2. 共有メッセージ添付アップロード
3. 個人メッセージ添付アップロード
4. 権限外取得拒否確認

## 3. 完了判定
- 全シナリオ pass
- 重大不具合なし
