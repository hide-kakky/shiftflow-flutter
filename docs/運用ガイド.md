# SHIFTFLOW Operations Guide v1.1

## 1. 日次運用
- 監視対象
  - Edge Functions ログ
  - `notification_dispatch_logs` 失敗件数
  - `audit_logs` の異常操作
  - 参加申請と招待受諾の滞留
- 定期作業
  - 失敗通知の再送確認
  - 停止/失効アカウントの棚卸し
  - 権限変更履歴レビュー

## 2. 参加運用
### 組織参加申請
1. `join_requests` を確認
2. 承認または拒否
3. 承認後に `memberships.status=active`

### 招待リンク
1. `organization_invites` を発行
2. 有効期限を確認
3. 使用済み / 失効の管理

## 3. ユニット運用
- ユニット作成
- ユニット移動
- ユニット責任者設定
- 親ユニット `manager` 継承範囲の確認

## 4. 事故対応
### 誤送信
1. 対象メッセージの文脈確認
2. 必要なら削除または訂正
3. 監査ログ確認

### 誤削除 / 誤停止
1. 対象名と実行者確認
2. 監査ログ確認
3. 復旧可能なら復旧

### 権限変更事故
1. 変更前後差分確認
2. 影響範囲確認
3. 必要ならロール差し戻し

## 5. 障害対応
### API障害
1. route と `code/reason` を確認
2. 組織 / ユニット文脈と権限を確認

### DB障害
1. migration 差分確認
2. RLS/関数差分確認
3. 必要なら修正 migration 追加

### 通知障害
1. `notification_dispatch_logs.status='failed'` を確認
2. `error_message` を分類
3. 再試行ジョブを実行

## 6. セキュリティ
- service role, PAT, 秘密鍵は公開しない
- RLS変更時は必ずテスト追加
- 危険ファイル拒否
- 個人情報、監査情報、招待情報の露出最小化
