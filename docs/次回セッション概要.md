## 直近で議論して固まった方向
- アカウントは自己登録とする
- 登録後に組織参加を行う
- 組織参加は `組織コード検索 -> 参加申請` と `招待リンク -> 自動参加` の2方式を有力とする
- `ユニット` を親子構造を持てる柔軟な所属単位とする
- `フォルダ` はユニット配下の共有・投稿単位とする
- ホームはダッシュボード件数中心ではなく、ユーザーが構成できる実データ中心画面へ寄せる
- タスクは現在地ユニットに依存せず全件表示する
- メッセージは現在地ユニット文脈で表示する
- モバイルのメッセージ画面では、ヘッダーに現在地ユニットを表示する
- メッセージ画面のタブでは `親ユニット`, `配下ユニット`, フォルダ群を扱う方向で整理中
- PC版ではパンくず表示を採用する
