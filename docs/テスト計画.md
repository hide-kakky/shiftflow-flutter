# SHIFTFLOW Testing Plan v1.1

## 1. テスト方針
- 目的: 新要件の情報設計、権限制御、参加導線、安全性を保証する。
- 範囲: DB/RLS, API, Flutter Unit/Widget/Integration, 通知, クロスプラットフォーム。

## 2. テストレベル
### 2.1 DB / RLS
- 組織越境アクセス拒否
- ユニット越境アクセス拒否
- フォルダ越境アクセス拒否
- DM越境アクセス拒否
- 親ユニット `manager` の継承確認
- `pending/suspended/revoked` の拒否

### 2.2 API
- bootstrap の状態別返却
- join / invite 導線
- `currentUnit` 切替
- shared / direct message の分離
- 添付取得権限
- 監査ログ記録の確認

### 2.3 Flutter
- Unit
  - Provider/Repository/DTO
  - 文脈管理
- Widget
  - 状態別画面表示
  - 段階表示 / 分割表示
  - エラー表示
- Integration
  - ログイン -> 組織参加 -> 利用開始
  - Home / Tasks / Messages / Admin の主要導線

### 2.4 通知
- 新規共有メッセージ
- 新規個人メッセージ
- 担当タスク作成
- 参加申請承認
- 招待受諾
- 重複防止・失敗時ログ確認

### 2.5 クロスプラットフォーム
- iOS / Android / Web で主要CUJをスモーク実行
- モバイル/PCで構成差はあるが、概念と用語が一致していること

## 3. 重点ケース
- 参加前 / 申請中 / active / suspended / revoked
- モバイル段階表示
- PC分割表示
- 上部文脈切替
- 1カラム全積み不採用
- 誤送信前確認
- 誤削除 / 誤停止確認
- 権限変更影響表示
- 危険ファイル拒否
